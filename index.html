Welcome to memedata’s documentation!¶

Memedata is bomdiabot’s service responsible for providing text and image data for memes. Everything is obtained via a REST API, which is described here.

The project’s repo can be found here.

API Summary¶

Resource Operation Description
Access Logout POST /auth/logout/access Revoke access token.
Create user POST /users Create user.
Delete Text DELETE /texts/(int:text_id) Remove text.
Delete user DELETE /users/(int:user_id) Delete user.
Get Text GET /texts/(int:text_id) Get text.
Get Texts GET /texts Get collection of texts.
Get user GET /users/(int:user_id) Get user.
Get users GET /users Get users.
Login POST /auth/login User login.
Refresh Logout POST /auth/logout/refresh Revoke refresh token.
Refresh token POST /auth/token/refresh Refresh access token.
Text creation POST /texts Create new text.
Update Text PUT /texts/(int:text_id) Update text.

API Details¶

POST /auth/logout/refresh¶

Revoke refresh token.

Example request:

POST /auth/logout/refresh HTTP/1.1
Host: example.com
Authorization: Bearer eyJ0eXAiOiJKV1QiLCJhb

Example response:

HTTP/1.1 204 NO CONTENT
Request Headers:
 
  • Authorization – refresh token of logged in user (required)
Status Codes:
  • 204 No Content – token revoked

GET /texts¶

Return collection of texts.

Example request:

GET /texts HTTP/1.1
Host: example.com
Accept: application/json
Authorization: Bearer eyJ0eXAiOiJKV1QiLCJhb

Example response:

HTTP/1.1 200 OK
Content-Type: application/json

{
    "texts": [
        {
            "content": "Bom dia!",
            "text_id": 1,
            "tags": [
                "bomdia"
            ],
            "updated_at": null,
            "created_at": "2018-09-15T22:53:26+00:00"
        },
        {
            "content": "Eu adoro as manhãs",
            "text_id": 32,
            "tags": [
                "jesus",
                "sexta"
            ],
            "updated_at": null,
            "created_at": "2018-09-15T22:53:26+00:00"
        },
    ],
    "offset": 2
}
Request Headers:
 
  • Authorization – access token of logged in user (required)
Query Parameters:
 
  • fields (string) – comma-separated list of fields to get for each text.
  • date_from (string) – only texts created after specified date (inclusive).
  • date_to (string) – only texts created before specified date.
  • any_tags (string) – texts with at least one tags in specified list.
  • all_tags (string) – texts only containing all specified tags.
  • no_tags (string) – texts only not containing any of specified tags.
  • offset (int) – pagination offset to start getting results
  • max_n_results (int) – maximum number of results to return.
Response Headers:
 
  • Content-Type – application/json
Status Codes:
  • 200 OK – texts found
Returns:

memedata.models.Text
